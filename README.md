## 🎯 项目概述

有一个庞大却常被忽视的群体——**三明治家庭**。

他们上有年迈父母需要照料，下有年幼子女需要养育，自己还要在职场打拼。这些家庭承载着三代人的健康重担，却往往陷入三大"健康黑洞"：

| 黑洞 | 真实困境 |
|------|----------|
| **健康数据四散** | 不只是病历——爸妈的血压血糖本、孩子的疫苗本和生长记录、妻子的经期数据、自己的健身体脂……全家人的健康数据散落在十几个APP、纸质本、微信聊天记录里，谁也拼不出完整的健康画像 |
| **照护无指导** | 血压高了要不要调药？新药和旧药会冲突吗？孩子发育是否达标？经期异常要不要就医？缺少一个懂全家情况的"人"来给建议 |
| **时间大出血** | 请假看病扣全勤，排队取药耗半天，医生说的转头就忘——而这些本可以通过提前准备和主动提醒大幅减少 |

**龙虾医生**正是为了解决这一社会痛点而诞生。我们不是又一个简单的健康记录工具，而是为龙虾安装了医疗版的"自动驾驶系统"——**从被动记录到主动预警，从单人管理到全家覆盖，从信息孤岛到科研联动**。让它成为每个家庭的：

- 🏠 **虚拟家庭医生** —— 全家人健康状况一目了然
- 🤝 **AI就诊助手** —— 诊前准备、诊中记录、诊后执行全程陪伴
- 🍽️ **家庭营养师** —— 兼顾全家健康需求的科学食谱
- 🔬 **医学情报员** —— 最新科研进展第一时间送达

## 🦞 二、龙虾医生能做什么？

### 🔧 基础能力（全家通用）

| 功能 | 说明 |
|------|------|
| **📸 拍照建档** | **问题：** 病历散落在各医院、各科室，纸质为主，换医院看病时病史答不全<br>**做法：** 拍病历/化验单/处方，AI自动OCR识别并结构化存储，自然语言关联家庭成员如"这是我妈的体检报告" |
| **👨‍👩‍👧‍👦 家庭成员管理** | **问题：** 全家人的健康信息散落各处，难以统一管理<br>**做法：** 自然语言添加成员，支持三代同堂家庭结构，成员间关联（如父子、夫妻），一人录入全家人受益 |
| **🔍 自然语言查询** | **问题：** 传统搜索需要精确关键词，查找健康记录困难<br>**做法：** 支持"我爸去年吃过什么药""宝宝上次发烧是什么时候"等自然语言查询，AI理解意图并精准检索，返回结构化答案 |
| **📋 就诊摘要生成** | **问题：** 换医院看病时，医生问病史，常常答不全或说不清<br>**做法：** 一键生成某人的病史摘要，按时间线整理关键诊断/手术/用药，适合打印或展示给新医生 |
| **🔒 隐私保护** | **承诺：** 仅限1对1私聊不在群聊泄露，数据本地存储物理隔离，用户完全控制自己的数据 |

### 👶 儿童专区

| 功能 | 说明 |
|------|------|
| **📈 生长曲线** | **问题：** 孩子长得好不好、发育是否正常，家长常常没有直观参考<br>**做法：** 记录身高体重头围数据，对照WHO儿童生长标准百分位，生成生长曲线图，发育偏离时（<P3或>P97）标红提醒 |
| **💉 疫苗管家** | **问题：** 新手爸妈常常困惑孩子现在该打什么疫苗<br>**做法：** 内置中国0-6岁一类+二类疫苗完整计划表，根据孩子出生日期自动计算当前应接种疫苗，记录接种时间/批次/机构，提前提醒避免漏种 |
| **🤒 生病日志** | **问题：** 孩子频繁生病，用药历史记不住，容易重复开药<br>**做法：** 记录每次生病的时间/症状/诊断，追踪用药历史和效果，频率统计发现健康规律 |

### 👩 女性/成人专区

| 功能 | 说明 |
|------|------|
| **🩸 经期记录** | **问题：** 女性需要追踪经期、症状，预测下次经期，规划备孕<br>**做法：** 记录周期/症状/流量，预测下次经期和排卵期，备孕期提醒，异常模式预警 |
| **💪 体脂/健身追踪** | **问题：** 减肥、健身需要科学追踪，凭感觉不可靠<br>**做法：** 记录体重/体脂率/BMI，生成趋势图，目标进度追踪 |
| **💊 用药冲突检测** | **问题：** 许多慢性病患者长期服用多种药物，但药物相互作用风险极高（如华法林+NSAIDs严重出血风险）<br>**做法：** 内置2000+常见药物相互作用规则库，新处方录入时自动检测在用药物冲突，同时检查慢性病禁忌，发现冲突立即红色警告并建议咨询医生 |

### 👴 老人专区

| 功能 | 说明 |
|------|------|
| **📊 慢病指标追踪** | **问题：** 从"生病再看医生"升级为"长期健康管理"<br>**做法：** 记录血压/血糖/尿酸等关键指标，自动生成趋势图表直观展示变化轨迹，标记异常值显示参考范围，支持按时间段/按成员查询对比 |
| **⏰ 复查/用药提醒** | **问题：** 医生常说"三个月后复查"，但工作忙碌、时间久远，很容易忘记<br>**做法：** 自动从病历OCR中识别"建议X月后复查"等关键信息，自动计算到期日期并创建提醒，跟踪提醒状态（待办/已完成/逾期） |
| **📝 用药清单** | **问题：** 老人用药多容易混淆，家人也不清楚具体吃什么药<br>**做法：** 整理老人在用所有药物，显示剂量/用法/禁忌，支持一键导出就医时展示给医生 |

### 🍽️ 全家功能

| 功能 | 说明 |
|------|------|
| **🍽️ 家庭饮食规划** | **问题：** 一个家庭里常有高血压老人（需低盐）、痛风患者（需低嘌呤）、成长中的孩子（需补钙）——饮食需求互相冲突<br>**做法：** 综合考虑所有家庭成员的慢性病和诊断，参考各疾病饮食规则取交集，优先保证儿童营养需求，生成具体每日/每周菜单建议 |
| **💰 医疗账本** | **问题：** 每年看病花了多少钱、花在哪儿了，大多数家庭没有清晰概念<br>**做法：** 自动汇总就诊费用和独立医疗支出，按成员/科室/时间段多维度统计，生成费用统计图表，支持自然语言查询 |
| **🚨 急救卡** | **问题：** 遇到紧急情况时，医护人员需要第一时间获取关键信息<br>**做法：** 一键生成家庭成员的急救信息卡（过敏/血型/慢性病/紧急联系人），可存手机锁屏 |
| **🌡️ 季节预警** | **问题：** 不同季节有不同健康风险，需要提前防范<br>**做法：** 根据季节和地区预警高发健康问题（如"春季到了注意过敏性鼻炎，提前备好开瑞坦"） |

### 🔬 高端功能：医学雷达

| 功能 | 说明 |
|------|------|
| **🔍 论文搜索** | **问题：** 家人得的病有没有新药、新疗法？普通人很难接触到最新医学研究<br>**做法：** 订阅疾病关键词自动追踪PubMed/medRxiv最新论文，使用LLM生成中文摘要，让普通人也能享受医学进步的红利 |
| **📡 订阅推送** | **问题：** 如何持续关注某个病/药的最新进展<br>**做法：** 关注某个病/药，每周推送最新进展，与家人病情联动 |
| **🔗 病情联动** | **问题：** 不知道该关注哪些研究方向<br>**做法：** 根据家人病史自动推荐关注的研究方向，家人相关研究特别标注 |

## ⚙️ 技术实现

### 系统架构

```
family-doctor/
├─ SKILL.md                  # ~130行，精简技能描述
├─ scripts/
│  ├─ init_db.py             # 家庭私有数据库初始化
│  ├─ query_db.py            # 15+查询函数，支持复杂健康查询
│  └─ charts.py              # 数据可视化（生长曲线、指标趋势等）
└─ references/
   ├─ schema.md              # 完整字段文档，确保数据规范性
   ├─ drug-interactions.md   # 用药冲突规则库，基于权威医学指南
   ├─ vaccine-schedule.md    # 中国疫苗接种计划，官方数据来源
   ├─ growth-standards.md    # WHO发育标准，国际权威参考
   └─ dietary-guidelines.md  # 饮食指南，融合中西医营养学

med-radar/
├─ SKILL.md                  # 医学科研雷达技能
├─ scripts/
│  ├─ init_db.py             # 订阅数据库初始化
│  ├─ query_db.py            # 订阅管理、摘要查询
│  └─ search_pubmed.py       # PubMed/medRxiv论文搜索
└─ references/
   ├─ sources.md             # 数据源详细说明
   └─ prompt-templates.md    # 摘要生成模板（周报/单篇解读）
```

### 技术特色

| 特色 | 说明 |
|------|------|
| **本地化优先** | 核心健康数据存储在家庭本地，确保隐私安全 |
| **模块化设计** | 八大功能相互独立又有机整合，便于迭代升级 |
| **AI能力** | 利用Claude的OCR、理解、推理能力处理医疗文本 |
| **开放接口** | 支持与主流健康设备、医院系统数据对接 |
| **技能联动** | family-doctor与med-radar自动协同，根据诊断建议研究订阅 |


### 数据库设计

采用SQLite3（含FTS5全文搜索）：

- `family_members` - 家庭成员基础信息
- `medical_records` - 病历记录（支持FTS搜索）
- `medications` - 用药记录
- `followups` - 复查提醒
- `health_metrics` - 健康指标
- `vaccinations` - 疫苗接种
- `growth_records` - 生长发育记录
- `expenses` - 医疗支出
- `health_notes` - 健康笔记
